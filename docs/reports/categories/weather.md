# Weather Intelligence — Report Family Recipe Matrix

## Persona & the fear it answers

**Who gets fired:** the **Operations Director / regional farm manager / DC scheduling lead** whose entire job is to *commit the next 7-14 days* — where crews go, when to spray, when to irrigate, when to harvest ahead of a front, when to protect a frost-sensitive block. Weather is the single input that overrules every other plan: one un-forecast frost kills a citrus block, one mis-timed spray drifts onto a neighbor and triggers a regulator, one wind event flattens the crew's harvest window, one storm strands trucks at the DC. The fear this family answers is blunt: *"Will the weather blow up my week, and can I prove to the CEO on Monday that I planned around it instead of getting surprised?"* Report.Farm is not a weather widget — it fuses public forecast physics with the farm's own satellite/terrain state and translates it into **do-this-by-Thursday operational orders** with a dollar and job-security frame ("protect the north block tonight or lose $340K of Gala").

```
lband_sar (flood)       │      Storm-Risk-Model          ├─ GNN Sensor-Fusion          │   window table       └─ push/SMS/    PDF + dashboard
Phenology / stand data ─┘      Spray-Window-Model        │    (station+sat+IoT)        ├─ $ impact (Financial)    email/SSE    + API (BUSINESS)
                               Terrain-Drainage          └─ Bayesian Confidence ───────┴─ Recommendation-LLM
                               Risk-Engine                                                  action orders
```

All 19 reports draw from one **Weather-Fusion** spine (public forecast + station obs downscaled to the parcel using the farm's own LST/terrain/NDVI state), then branch into the peril- or accumulation-specific primitive. Observed-state peril extents (flood via SAR, heat via LST, rainfall via GPM) are T1/T2 today; **forward forecasts are model-inferred (T3)** and honestly labeled as such.
