The autonomous operating layer for farms and the supply chains that depend on them.

Report.Farm watches land, crops, water and assets across single fields to entire supplier portfolios — explaining what changed, what it costs, and what to do, as scheduled reports and urgent alerts. This blueprint maps every surface, its sections and elements, the people who use it, and the journeys they take. Built as a thin vertical on the AlphaGeo geospatial engine.

01 — Orientation

How the product is organized

Report.Farm is one multi-tenant application with four zones. A public marketing site tells the story and captures interest. An authentication gate (passcode + login) scopes every session to a tenant. The operating console is where work happens — a set of role-gated surfaces sharing one shell. A field & map zone handles on-the-ground and geospatial views. Everything the console shows is served by the /api/farm/* API over the farm digital twin.

Tenancy model

  • Buyer = tenantiam.tenant
  • Supplier / grower orgfarm.supplier
  • Sourcing regionfarm.sourcing_region
  • Farm → parcel → zonefarm.*

Isolation & security

  • Row-level security, every tableRLS
  • Role → permission RBACfarm.yaml
  • Passcode access gateprod
  • Zero cross-tenant leakverified

The monitoring loop

  • Onboard farm + zone intent
  • Satellite pass → analysisP2
  • Observation → signal → alertP2
  • Report + portfolio rollup

06 — User stories

What each persona needs to do

Grouped by persona, in the "as a … I want … so that …" form, each with its key acceptance signal.

Buyer / Portfolio Lead

As a buyer, I want one portfolio view of every supplier's risk so that I can see sourcing exposure without chasing individual farms.Dashboard shows supplier count, portfolio risk, yield-at-risk and revenue-at-risk from a single rollup call.
As a buyer, I want suppliers ranked by risk so that I act on the worst exposure first.Supplier table sorts by peak risk; each row carries a labelled severity pill.
As a buyer, I want urgent disruption alerts so that I can re-source or hedge before a bad season becomes a broken contract.Active-disruptions feed surfaces threshold crossings; empty-state is honest until the first pass.
As a buyer, I want to generate an executive report for a period so that I can brief my team with evidence.Report generates from live twin data with data-quality notes; no fabricated signals.

Farm Operations

As ops, I want to onboard a farm by drawing or importing its boundary so that monitoring can start in minutes.Boundary saved; area auto-computed; self-intersecting shapes rejected with a clear inline error.
As ops, I want to bulk-import many supplier farms so that a large portfolio onboards without manual drawing.Shapefile/KML/CSV import creates farms + parcels per feature.
As ops, I want to tag zones with intent (irrigation, barn, wetland) so that alerts respect what each zone is for.Zone stores intent JSON; a standing-water reading in a "no standing water" zone becomes a critical alert.
As ops, I want to acknowledge and triage alerts so that the team isn't re-alerted on known issues.Alert status moves open → ack; dedup prevents replay double-fire.

Buyer Admin

As an admin, I want tenant-isolated data so that no buyer ever sees another's suppliers.Row-level security on every farm table; verified zero cross-tenant leakage.
As an admin, I want to manage users, roles and invites so that each teammate gets the right surfaces.Role grants map to permission bundles; users land on their primary surface.

Grower & Field

As a grower, I want to see only my own farms and their findings so that my data stays private within the network.Grower portal scopes to the grower's identity; no portfolio-wide access.
As a field agronomist, I want to check in at a farm and log what I find so that ground truth sharpens the satellite signal.Geofenced check-in validated against the boundary; photos + notes attach to the farm.
